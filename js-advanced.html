비동기 프로그래밍, API 통신, 프레임워크까지 JavaScript의 모든 것을 마스터하세요

JavaScript란 무엇인가요?

JavaScript는 웹의 마법사입니다

JavaScript는 웹 페이지를 살아 움직이게 하는 프로그래밍 언어입니다. HTML이 웹 페이지의 뼈대를 만들고, CSS가 디자인을 입힌다면, JavaScript는 웹 페이지에 생명력과 상호작용을 불어넣습니다.

실제 예시로 이해하기

JavaScript 없이는 웹페이지가 어떻게 보일까요?

JavaScript 없는 웹페이지

상품 수량: 1 (변경 불가)

총 가격: 10,000원 (계산 불가)

클릭해도 아무 반응 없음

JavaScript 있는 웹페이지

온라인 쇼핑몰

1
총 가격: 10,000원

인사이트: JavaScript가 있으면 버튼이 반응하고, 수량을 변경할 수 있으며, 실시간으로 가격이 계산됩니다!

쉬운 비유로 이해하기

웹페이지를 사람이라고 생각해보세요:

  • HTML = 뼈와 장기 (기본 구조)
  • CSS = 옷과 화장 (외모와 스타일)
  • JavaScript = 뇌와 근육 (생각하고 움직이는 능력)

JavaScript가 없으면 아무리 예쁘게 꾸며진 웹페이지라도 마네킹처럼 움직일 수 없습니다!

변수와 상수

변수란 데이터를 담는 상자입니다

변수는 값을 저장하고 나중에 사용할 수 있도록 해주는 메모리 공간입니다. 마치 이름표가 붙은 상자에 물건을 넣어두는 것과 같습니다.

변수 선언 방법 3가지

1. let (추천)

let userName = "김철수";
let age = 25;
let isStudent = true;

// 나중에 값 변경 가능
userName = "이영희";
age = 30;
장점:
  • 값 변경 가능
  • 블록 스코프
  • 중복 선언 방지

2. const (상수)

const PI = 3.14159;
const siteName = "WebLearn";
const maxUsers = 1000;

// 값 변경 시도하면 오류 발생
// PI = 3.14; // 오류!
사용 상황:
  • 변하지 않는 값
  • 설정값, 상수
  • 안전한 코딩

3. var (사용 지양)

var oldStyle = "구식 방법";
var count = 10;

// 문제가 많아서 사용하지 않음
문제점:
  • 스코프 문제
  • 중복 선언 허용
  • 예상치 못한 오류

변수 이름 짓기 규칙

올바른 예시

let userName = "홍길동";
let userAge = 25;
let isLoggedIn = true;
let totalPrice = 50000;
let _privateVar = "비공개";

잘못된 예시

변수 이름 짓기 팁

  • camelCase 사용: userName, totalPrice, isLoggedIn
  • 의미 있는 이름: a, b, x 대신 userName, age, price
  • 동사 + 명사: getUserName(), calculateTotal()
  • 불린 변수: is, has, can으로 시작 (isValid, hasPermission)

데이터 타입

JavaScript의 8가지 데이터 타입

JavaScript는 값의 종류에 따라 다른 데이터 타입을 가집니다. 각 타입을 이해하면 더 효율적인 프로그래밍이 가능합니다.

기본 타입 (Primitive Types)

1. 문자열 (String)

let name = "홍길동";
let message = '안녕하세요!';
let template = `저는 ${name}입니다.`; // 템플릿 리터럴

// 문자열 연결
let fullMessage = "안녕하세요! " + name + "님";
let templateMessage = `안녕하세요! ${name}님`;

// 문자열 길이
console.log(name.length); // 3

// 문자열 메서드
console.log(name.toUpperCase()); // 대문자로
console.log(name.toLowerCase()); // 소문자로

2. 숫자 (Number)

console.log(pi.toFixed(2)); // "3.14"

// typeof 연산자 사용
console.log(typeof "hello"); // "string"

console.log(typeof null); // "object" (주의!)
console.log(typeof []); // "object"

console.log(Array.isArray({})); // false

// 실무에서 타입 체크
function processData(data) {
    if (typeof data === "string") {
        return data.toUpperCase();
    } else if (typeof data === "number") {
        return data * 2;
    } else {
        return "알 수 없는 타입";
    }
}

연산자

산술 연산자

let a = 10;
let b = 3;

console.log(a + b); // 13 (덧셈)
console.log(a - b); // 7  (뺄셈)
console.log(a * b); // 30 (곱셈)
console.log(a / b); // 3.333... (나눗셈)
console.log(a % b); // 1  (나머지)
console.log(a ** b); // 1000 (거듭제곱)

// 증감 연산자
let count = 5;
count++; // count = count + 1
console.log(count); // 6

count--; // count = count - 1
console.log(count); // 5

비교 연산자

let x = 5;
let y = "5";

// 동등 비교
console.log(x == y);  // true (값만 비교)
console.log(x === y); // false (값과 타입 모두 비교)

// 부등 비교
console.log(x != y);  // false
console.log(x !== y); // true

// 크기 비교
console.log(10 > 5);   // true
console.log(10 < 5);   // false
console.log(10 >= 10); // true
console.log(10 <= 5);  // false

// 실무 예시
let userAge = 20;
let isAdult = userAge >= 18;
let canDrink = userAge >= 21;

논리 연산자

let isLoggedIn = true;
let hasPermission = false;
let isAdmin = true;

// AND 연산자 (&&)
console.log(isLoggedIn && hasPermission); // false
console.log(isLoggedIn && isAdmin); // true

// OR 연산자 (||)
console.log(isLoggedIn || hasPermission); // true
console.log(hasPermission || isAdmin); // true

// NOT 연산자 (!)
console.log(!isLoggedIn); // false
console.log(!hasPermission); // true

// 실무 활용
let canEditPost = isLoggedIn && (isAdmin || hasPermission);
let showLoginButton = !isLoggedIn;

할당 연산자

let score = 100;

// 기본 할당
score = 90;

// 복합 할당
score += 10; // score = score + 10; (100)
score -= 5;  // score = score - 5;  (95)
score *= 2;  // score = score * 2;  (190)
score /= 2;  // score = score / 2;  (95)
score %= 10; // score = score % 10; (5)

// 문자열에도 사용 가능
let message = "안녕";
message += "하세요!"; // "안녕하세요!"

실무 예시: 쇼핑몰 계산기

// 상품 정보
let productPrice = 25000;
let quantity = 3;
let discountRate = 0.1; // 10% 할인

// 계산
let subtotal = productPrice * quantity;
let discountAmount = subtotal * discountRate;
let finalTotal = subtotal - discountAmount;

// 배송비 계산
let shippingFee = finalTotal >= 50000 ? 0 : 3000;
let grandTotal = finalTotal + shippingFee;

// 결과 출력
console.log(`상품 가격: ${productPrice.toLocaleString()}원`);
console.log(`수량: ${quantity}개`);
console.log(`소계: ${subtotal.toLocaleString()}원`);
console.log(`할인: ${discountAmount.toLocaleString()}원`);
console.log(`배송비: ${shippingFee.toLocaleString()}원`);
console.log(`최종 총액: ${grandTotal.toLocaleString()}원`);

조건문

if 조건문

특정 조건이 참일 때만 코드를 실행하고 싶을 때 사용합니다.

let age = 20;

// 기본 if문
if (age >= 18) {
    console.log("성인입니다.");
}

// if-else문
if (age >= 18) {
    console.log("성인입니다.");
} else {
    console.log("미성년자입니다.");
}

// if-else if-else문
if (age < 13) {
    console.log("어린이입니다.");
} else if (age < 18) {
    console.log("청소년입니다.");
} else if (age < 65) {
    console.log("성인입니다.");
} else {
    console.log("노인입니다.");
}

실무 프로젝트: 로그인 시스템

// 사용자 정보
let users = [
    { username: "admin", password: "1234", role: "admin" },
    { username: "user1", password: "pass", role: "user" },
    { username: "user2", password: "secret", role: "user" }
];

// 로그인 함수
function login(username, password) {
    // 사용자 찾기
    let user = null;
    for (let i = 0; i < users.length; i++) {
        if (users[i].username === username) {
            user = users[i];
            break;
        }
    }
    
    // 로그인 검증
    if (user === null) {
        return "존재하지 않는 사용자입니다.";
    } else if (user.password !== password) {
        return "비밀번호가 틀렸습니다.";
    } else if (user.role === "admin") {
        return "관리자로 로그인되었습니다.";
    } else {
        return "일반 사용자로 로그인되었습니다.";
    }
}

// 테스트
console.log(login("admin", "1234")); // "관리자로 로그인되었습니다."
console.log(login("user1", "pass")); // "일반 사용자로 로그인되었습니다."
console.log(login("user1", "wrong")); // "비밀번호가 틀렸습니다."
console.log(login("nobody", "pass")); // "존재하지 않는 사용자입니다."
