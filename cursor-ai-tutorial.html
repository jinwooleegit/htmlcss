Cursor AI란?

AI 기반 코드 에디터

Cursor는 GPT-4를 기반으로 한 AI 코딩 도구로, 자연어로 요청하면 자동으로 코드를 생성하고 수정해주는 혁신적인 에디터입니다.

AI와 함께 첫 웹페이지 만들기

프롬프트 예시

💬 AI에게 이렇게 요청해보세요:

"개인 포트폴리오 웹사이트의 메인 페이지를 만들어줘. 
다음 요소들을 포함해줘:
- 상단 네비게이션 바
- 히어로 섹션 (이름과 직업)
- 자기소개 섹션
- 포트폴리오 섹션
- 연락처 섹션
- 푸터

색상은 파란색 계열로 하고, 현대적이고 깔끔한 디자인으로 해줘."

좋은 프롬프트 작성법

  • 구체적으로: "버튼" 보다는 "파란색 둥근 모서리 버튼"
  • 구조 명시: 원하는 섹션들을 나열
  • 스타일 지정: 색상, 분위기, 레이아웃 스타일
  • 기능 설명: 클릭하면 어떤 동작을 할지

단계별 웹사이트 제작 과정

1단계: HTML 구조 생성

💬 프롬프트:
"index.html 파일을 만들어줘. 
포트폴리오 웹사이트의 기본 구조를 만들어줘:

1. header - 네비게이션
2. main 
   - hero 섹션
   - about 섹션  
   - portfolio 섹션
   - contact 섹션
3. footer

시맨틱 태그를 사용하고 SEO에 최적화해줘."

AI 활용 팁

  • 큰 구조부터 만들고 세부사항은 나중에
  • 파일별로 나누어서 요청
  • 생성된 코드를 확인하고 수정 요청

2단계: CSS 스타일링

💬 프롬프트:
"style.css 파일을 만들어서 방금 만든 HTML에 스타일을 적용해줘:

- 전체적으로 모던하고 미니멀한 디자인
- 색상: 메인 #2563eb, 서브 #64748b, 배경 #f8fafc
- 폰트: 깔끔한 sans-serif
- 네비게이션은 고정형으로
- 히어로 섹션은 전체 화면 높이
- 각 섹션마다 적절한 여백
- 반응형으로 모바일에서도 잘 보이게"

스타일링 요청 팁

  • 색상 코드를 구체적으로 명시
  • 레이아웃 방식 지정 (Flexbox, Grid)
  • 반응형 디자인 요구사항 포함
  • 애니메이션이나 효과 원하면 명시

3단계: JavaScript 인터랙션

💬 프롬프트:
"script.js 파일을 만들어서 다음 기능들을 추가해줘:

1. 스크롤할 때 네비게이션 배경 변경
2. 네비게이션 메뉴 클릭하면 해당 섹션으로 부드럽게 이동
3. 포트폴리오 이미지에 호버 효과
4. 연락처 폼 유효성 검사
5. 모바일에서 햄버거 메뉴 동작

모든 기능은 부드럽고 자연스럽게 동작하게 해줘."

인터랙션 개발 팁

  • 원하는 동작을 구체적으로 설명
  • 사용자 경험(UX)을 고려한 요청
  • 브라우저 호환성 고려 요청
  • 성능 최적화도 함께 요청

효과적인 AI 프롬프트 작성법

고급 프롬프트 예시

🎯 구체적이고 상세한 요청:

"현재 포트폴리오 섹션을 개선해줘:
- 3x2 그리드 레이아웃으로 6개 프로젝트 표시
- 각 카드에 이미지, 제목, 설명, 기술 스택 태그
- 호버 시 카드가 살짝 위로 올라가는 애니메이션
- 클릭하면 프로젝트 상세 페이지로 이동
- 카드 디자인은 그림자와 둥근 모서리로 모던하게
- 모바일에서는 1열로 변경되게
- 로딩 시 카드들이 순차적으로 나타나는 효과"

명확한 목표

무엇을 만들고 싶은지 구체적으로 설명

시각적 디테일

색상, 크기, 레이아웃, 애니메이션 명시

기능적 요구사항

클릭, 호버, 스크롤 등 인터랙션 설명

반응형 고려

다양한 화면 크기에서의 동작 방식

AI와 함께하는 문제 해결

문제 해결 프롬프트

🔧 문제 상황별 요청 방법:

📱 반응형 문제:
"모바일에서 네비게이션이 깨져보여. 
화면 크기 768px 이하에서 햄버거 메뉴로 바뀌게 수정해줘."

🎨 디자인 문제:
"히어로 섹션의 텍스트가 배경 이미지와 구분이 안 돼. 
텍스트에 반투명한 배경을 추가하거나 그림자 효과를 줘."

⚡ 성능 문제:
"페이지 로딩이 너무 느려. 이미지 최적화와 CSS/JS 압축을 해줘."

🐛 기능 오류:
"연락처 폼에서 이메일 전송이 안 돼. 
PHP나 JavaScript로 실제 작동하는 폼을 만들어줘."

효과적인 디버깅 방법

  • 문제 상황을 구체적으로 설명
  • 에러 메시지가 있다면 함께 제공
  • 어떤 브라우저/기기에서 발생하는지 명시
  • 원하는 결과를 명확히 설명

실제 프로젝트: 카페 웹사이트 만들기

프로젝트 전체 플랜

☕ 카페 웹사이트 제작 계획:

1단계: 기본 구조
"카페 웹사이트를 만들어줘. 
페이지: 홈, 메뉴, 매장소개, 연락처
따뜻하고 아늑한 분위기의 갈색 톤 디자인"

2단계: 메뉴 페이지
"메뉴 페이지에 커피, 디저트, 베이커리 카테고리로 
상품들을 카드 형태로 표시해줘. 
각 상품에 이미지, 이름, 가격, 설명 포함"

3단계: 예약 시스템
"매장소개 페이지에 테이블 예약 폼을 추가해줘.
날짜, 시간, 인원수, 연락처를 입력받고
달력 위젯으로 날짜 선택하게 해줘"

4단계: 지도 연동
"연락처 페이지에 Google Maps 연동하고
매장 위치, 운영시간, 전화번호 표시해줘"

프로젝트 성공 요소

  • 단계별로 나누어 진행
  • 각 단계마다 테스트 및 검토
  • 사용자 관점에서 기능 확인
  • 모바일 최적화 필수
  • 실제 데이터로 테스트

웹사이트 최적화 및 마무리

최적화 체크리스트

🚀 최종 최적화 요청:

"웹사이트를 최종 검토하고 다음 사항들을 최적화해줘:

1. SEO 최적화
- 각 페이지에 적절한 title, meta description
- 이미지에 alt 속성 추가
- 구조화된 데이터 마크업

2. 성능 최적화  
- 이미지 압축 및 WebP 형식 사용
- CSS/JS 파일 압축
- 불필요한 코드 제거

3. 접근성 개선
- 키보드 네비게이션 지원
- 적절한 색상 대비
- 스크린 리더 지원

4. 브라우저 호환성
- 구형 브라우저에서도 작동하게
- CSS fallback 추가"
