제목 태그 사용 가이드라인

  • h1: 페이지의 메인 주제 (SEO에 중요)
  • h2: 주요 섹션 구분
  • h3-h6: 하위 구조 표현
  • 제목 레벨을 건너뛰지 마세요 (h1 → h3 금지)
실제 블로그 포스트 구조 예시:
<article>
    <h1>웹 개발 완전정복 가이드</h1>        <!-- 메인 제목 -->
    
    <h2>1. 프론트엔드 개발</h2>            <!-- 주요 섹션 -->
    <h3>HTML 기초</h3>                    <!-- 하위 섹션 -->
    <h4>태그의 종류</h4>                  <!-- 세부 항목 -->
    <h4>속성 활용법</h4>
    
    <h3>CSS 스타일링</h3>
    <h4>선택자 사용법</h4>
    <h4>레이아웃 기법</h4>
    
    <h2>2. 백엔드 개발</h2>              <!-- 다음 주요 섹션 -->
    <h3>서버 기초</h3>
    <h3>데이터베이스</h3>
</article>
SEO 최적화 팁:
  • h1은 페이지당 하나만 사용하세요
  • 키워드를 자연스럽게 제목에 포함시키세요
  • 제목만 읽어도 내용을 알 수 있게 작성하세요
  • 50자 이내로 간결하게 작성하세요

이미지 완전 활용법

<!-- 기본 이미지 -->
<img src="photo.jpg" alt="사진 설명" width="300" height="200">

<!-- 반응형 이미지 -->
<img src="photo-large.jpg" 
     srcset="photo-small.jpg 480w, photo-medium.jpg 800w, photo-large.jpg 1200w"
     sizes="(max-width: 480px) 100vw, (max-width: 800px) 50vw, 25vw"
     alt="반응형 이미지">

<!-- 이미지 맵 (클릭 가능한 영역) -->
<img src="worldmap.jpg" alt="세계지도" usemap="#worldmap">
<map name="worldmap">
    <area shape="rect" coords="0,0,100,100" href="korea.html" alt="한국">
    <area shape="circle" coords="200,200,50" href="usa.html" alt="미국">
</map>

<!-- 이미지와 캡션 -->
<figure>
    <img src="sunset.jpg" alt="아름다운 석양">
    <figcaption>제주도에서 본 아름다운 석양</figcaption>
</figure>

이미지 최적화 팁

  • alt 속성: 항상 의미 있는 설명 작성
  • 파일 크기: 웹용으로 압축 (보통 100KB 이하)
  • 파일 형식: JPEG(사진), PNG(투명배경), WebP(최신)
  • 로딩 최적화: loading="lazy" 속성 활용
실무에서 자주 사용하는 이미지 패턴:
1. 프로필 이미지 (SNS, 쇼핑몰)
<div class="profile-section">
    <img src="profile.jpg" 
         alt="김철수 프로필 사진" 
         width="100" 
         height="100"
         loading="lazy">
    <div class="profile-info">
        <h3>김철수</h3>
        <p>웹 개발자</p>
    </div>
</div>
2. 상품 갤러리 (이커머스)
<div class="product-gallery">
    <figure>
        <img src="product-main.jpg" 
             alt="아이폰 15 Pro 메인 이미지"
             id="main-image">
        <figcaption>아이폰 15 Pro - 티타늄 블루</figcaption>
    </figure>
    
    <div class="thumbnail-list">
        <img src="thumb1.jpg" alt="정면 모습" onclick="changeMain(this)">
        <img src="thumb2.jpg" alt="측면 모습" onclick="changeMain(this)">
        <img src="thumb3.jpg" alt="후면 모습" onclick="changeMain(this)">
    </div>
</div>
3. 배너/히어로 이미지 (랜딩 페이지)
<header class="hero-section">
    <picture>
        <source media="(max-width: 768px)" srcset="hero-mobile.webp">
        <source media="(max-width: 1024px)" srcset="hero-tablet.webp">
        <img src="hero-desktop.webp" 
             alt="웹 개발 교육 과정 - 3개월 완성"
             class="hero-image">
    </picture>
    
    <div class="hero-content">
        <h1>3개월 만에 웹 개발자 되기</h1>
        <p>HTML, CSS, JavaScript 완전 정복</p>
        <button>지금 시작하기</button>
    </div>
</header>
접근성 고려사항:
좋은 예시:
<img src="chart.png" alt="2024년 매출 증가율 - 1분기 15%, 2분기 23%, 3분기 18% 상승">

시각장애인도 차트의 핵심 정보를 이해할 수 있습니다.

나쁜 예시:
<img src="chart.png" alt="차트">

너무 간단해서 실제 정보를 전달하지 못합니다.

성능 최적화 고급 기법:
  • Lazy Loading: loading="lazy"로 스크롤 시 로딩
  • WebP 포맷: JPEG보다 25-35% 작은 파일 크기
  • Picture 요소: 디바이스별 최적화된 이미지 제공
  • 이미지 압축: TinyPNG, ImageOptim 등 도구 활용
  • CDN 활용: 전 세계 빠른 이미지 로딩

선호사항

폼 구성 요소 설명

  • fieldset + legend: 관련 입력 요소들을 그룹화
  • label: 입력 요소의 설명 (for 속성으로 연결)
  • input type="text": 일반 텍스트 입력
  • input type="email": 이메일 형식 검증
  • input type="password": 비밀번호 (숨김 표시)
  • input type="radio": 단일 선택
  • input type="checkbox": 다중 선택
  • select + option: 드롭다운 목록
  • textarea: 여러 줄 텍스트 입력

멀티미디어 활용

오디오와 비디오

<!-- 오디오 -->
<audio controls>
    <source src="music.mp3" type="audio/mpeg">
    <source src="music.ogg" type="audio/ogg">
    브라우저가 오디오를 지원하지 않습니다.
</audio>

<!-- 비디오 -->
<video width="400" height="300" controls poster="thumbnail.jpg">
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.webm" type="video/webm">
    <track src="subtitles.vtt" kind="subtitles" srclang="ko" label="한국어">
    브라우저가 비디오를 지원하지 않습니다.
</video>

<!-- YouTube 임베드 -->
<iframe width="560" height="315" 
        src="https://www.youtube.com/embed/VIDEO_ID" 
        title="YouTube video player" 
        frameborder="0" 
        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
        allowfullscreen>
</iframe>

멀티미디어 최적화 팁

  • 여러 포맷 제공: 브라우저 호환성을 위해
  • controls 속성: 재생 컨트롤 표시
  • poster 속성: 비디오 썸네일 이미지
  • 자막 지원: track 태그로 접근성 향상
  • 적절한 크기: 파일 크기와 품질의 균형
