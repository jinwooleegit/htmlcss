웹 호스팅 기초 이해

웹 호스팅이란?

웹 호스팅은 여러분의 웹사이트 파일들을 인터넷에 연결된 서버에 저장하여, 전 세계 사람들이 접근할 수 있게 하는 서비스입니다.

웹 호스팅 구성 요소

웹사이트 구조:

📁 내 컴퓨터 (로컬)
├── index.html
├── style.css
├── script.js
└── images/

⬇️ 업로드

🌐 웹 서버 (호스팅)
├── index.html      ← 방문자가 접근
├── style.css       ← 스타일 적용
├── script.js       ← 기능 실행
└── images/         ← 이미지 표시

서버

24시간 켜져있는 컴퓨터로, 웹사이트 파일을 저장하고 방문자들에게 제공

도메인

웹사이트의 주소 (예: www.mywebsite.com)

DNS

도메인 이름을 서버의 실제 주소(IP)로 변환하는 시스템

호스팅 서비스 종류

호스팅 종류 가격 난이도 적합한 용도 추천 서비스
무료 호스팅 무료 쉬움 연습, 포트폴리오 GitHub Pages, Netlify
공유 호스팅 월 3,000-10,000원 쉬움 개인 웹사이트, 블로그 카페24, 가비아, 호스팅케이알
VPS 호스팅 월 10,000-50,000원 중간 중소규모 비즈니스 AWS EC2, DigitalOcean
전용 서버 월 100,000원+ 어려움 대규모 웹사이트 전문 호스팅 업체

초보자 추천 경로

1단계: GitHub Pages (무료) → 2단계: 카페24/가비아 공유호스팅 → 3단계: AWS/DigitalOcean VPS

GitHub Pages로 무료 배포하기

1. GitHub 저장소 생성

1. GitHub.com 접속 후 로그인
2. 'New repository' 클릭
3. Repository name: 'my-website'
4. Public 선택
5. 'Create repository' 클릭
1

파일 업로드

HTML, CSS, JS 파일들을 저장소에 업로드

2

Pages 설정

Settings → Pages → Source를 'Deploy from a branch'로 설정

3

배포 완료

https://username.github.io/my-website 주소로 접속 가능

도메인 구입 가이드

도메인 선택 기준

좋은 도메인의 조건:

✅ 기억하기 쉬운 이름
✅ 브랜드와 관련된 이름  
✅ 15자 이내로 간결하게
✅ .com 확장자 우선
✅ 특수문자 사용 금지

예시:
myportfolio.com     ← 좋음
my-awesome-portfolio-website.co.kr  ← 너무 김
.com (연 15,000원) .co.kr (연 20,000원) .net (연 18,000원) .org (연 16,000원)
국내 업체
  • 가비아 (gabia.com)
  • 후이즈 (whois.co.kr)
  • 카페24 (cafe24.com)
해외 업체
  • GoDaddy (godaddy.com)
  • Namecheap (namecheap.com)
  • Google Domains

실제 배포 단계별 가이드

1. 저장소 준비

GitHub에 public 저장소를 생성하고 웹사이트 파일들을 업로드합니다.

2. Pages 활성화

Settings → Pages에서 Source를 'Deploy from a branch'로 설정합니다.

3. 브랜치 선택

main 브랜치의 root 폴더를 선택하고 Save를 클릭합니다.

4. 배포 확인

몇 분 후 제공된 URL로 접속하여 웹사이트가 정상 작동하는지 확인합니다.

1. 호스팅 업체 선택

카페24, 가비아 등에서 웹호스팅 상품을 구매합니다.

2. FTP 접속 정보 확인

호스팅 업체에서 제공하는 FTP 서버 주소, 아이디, 비밀번호를 확인합니다.

3. 파일 업로드

FileZilla 등의 FTP 프로그램으로 웹사이트 파일들을 서버에 업로드합니다.

4. 도메인 연결

구입한 도메인을 호스팅 서버에 연결하고 DNS 설정을 완료합니다.

1. VPS 서버 구매

AWS EC2, DigitalOcean 등에서 VPS 인스턴스를 생성합니다.

2. 서버 설정

Ubuntu/CentOS 설치 후 웹서버(Apache/Nginx) 및 필요한 소프트웨어를 설치합니다.

3. 웹사이트 배포

Git clone 또는 SCP로 웹사이트 파일들을 서버에 업로드합니다.

4. 보안 설정

SSL 인증서 설치, 방화벽 설정, 정기 백업 등 보안 조치를 실행합니다.
