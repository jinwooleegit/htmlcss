변수란 무엇인가요?

변수 = 데이터를 저장하는 상자

변수 (Variable)

변하는 수, 값을 저장하고 나중에 사용할 수 있는 저장소

예: 게임에서 점수, 사용자 이름, 장바구니 개수 등을 저장
변수명 (Variable Name)

상자에 붙이는 이름표

예: score, userName, itemCount 등 의미 있는 이름 사용
값 (Value)

상자 안에 들어있는 실제 데이터

예: 숫자, 문자, 참/거짓 등 다양한 형태의 데이터

실제 예시로 이해하기

변수를 사용하는 상황

변수 없이 코딩
alert("안녕하세요, 김철수님!");
console.log("김철수님의 점수: 85점");
alert("김철수님, 환영합니다!");

❌ 이름을 바꾸려면 모든 곳을 수정해야 함

변수 사용
let userName = "김철수";
let score = 85;

alert("안녕하세요, " + userName + "님!");
console.log(userName + "님의 점수: " + score + "점");
alert(userName + "님, 환영합니다!");

변수만 바꾸면 모든 곳이 자동으로 변경됨

인사이트: 변수를 사용하면 코드를 효율적으로 관리하고 재사용할 수 있습니다!

쉬운 비유로 이해하기

변수 = 사물함

  • 사물함 번호 = 변수명 (예: locker1, myBox)
  • 사물함 안의 물건 = 값 (예: 책, 운동화, 도시락)
  • 물건 바꾸기 = 값 변경 (새로운 물건으로 교체)

사물함 번호를 알면 언제든지 물건을 넣고 빼고 바꿀 수 있어요!

변수 선언 방법

JavaScript에서 변수를 만드는 3가지 방법

1

let - 변경 가능한 변수

추천 
let age = 25;
let name = "홍길동";
let isStudent = true;

// 나중에 값 변경 가능
age = 26;
name = "김철수";
장점: 값 변경 가능, 블록 스코프
사용 시기: 값이 바뀔 수 있는 데이터
2

const - 변경 불가능한 상수

상수 
const PI = 3.14159;
const SITE_NAME = "WebLearn";
const API_URL = "https://api.example.com";

// 오류 발생: 값 변경 불가
// PI = 3.14; // TypeError 발생
✅ 장점: 값 변경 방지, 안정성 높음
🎯 사용 시기: 고정된 값, 설정값
3

var - 구형 변수 선언

권장 안함 
var oldVariable = "구형 방식";
var count = 10;

// 같은 이름 재선언 가능 (문제 발생 가능)
var count = 20;
❌ 단점: 재선언 가능, 스코프 문제
🎯 사용 시기: 특별한 경우가 아니면 사용 안함

변수 선언 방법 비교

구분 let const var
값 변경 ✅ 가능 ❌ 불가능 ✅ 가능
재선언 ❌ 불가능 ❌ 불가능 ✅ 가능
스코프 블록 스코프 블록 스코프 함수 스코프
권장도 ✅ 권장 ✅ 권장 ❌ 비권장

데이터 타입

JavaScript의 주요 데이터 타입

🔤

문자열 (String)

텍스트 
let name = "홍길동";
let message = '안녕하세요!';
let template = `안녕하세요, ${name}님!`;

console.log(name);        // 홍길동
console.log(message);     // 안녕하세요!
console.log(template);    // 안녕하세요, 홍길동님!
특징:
  • "큰따옴표", '작은따옴표', `백틱` 사용
  • 백틱은 템플릿 리터럴 (변수 삽입 가능)
  • 문자열 연결: + 연산자 사용
🔢

숫자 (Number)

수치 
let age = 25;           // 정수
let height = 175.5;     // 소수
let negative = -10;     // 음수
let large = 1000000;    // 큰 수

console.log(age + 5);   // 30
console.log(height * 2); // 351
console.log(10 / 3);    // 3.3333333333333335
특징:
  • 정수와 소수를 구분하지 않음
  • 사칙연산 가능: +, -, *, /
  • 특수값: Infinity, -Infinity, NaN
✅

불린 (Boolean)

참/거짓 
let isStudent = true;
let isTeacher = false;
let hasPermission = true;

console.log(isStudent);     // true
console.log(age > 18);      // true (나이가 18보다 큼)
console.log(5 < 3);         // false
특징:
  • 오직 true 또는 false 값만 가짐
  • 조건문에서 자주 사용
  • 비교 연산 결과로 생성
❓

undefined와 null

빈 값 
let notDefined;           // undefined
let empty = null;         // null

console.log(notDefined);  // undefined
console.log(empty);       // null

// 차이점
console.log(typeof notDefined); // "undefined"
console.log(typeof empty);      // "object"
차이점:
  • undefined: 값이 할당되지 않음
  • null: 의도적으로 비어있음을 표시

typeof 연산자로 데이터 타입 확인하기

console.log(typeof "안녕하세요");  // "string"
console.log(typeof 42);          // "number"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" (특별한 경우)

실습: 변수와 데이터 타입 사용하기

실습 1: 개인 정보 저장하기

1

변수 만들기

// 개인 정보 변수 만들기
let name = "김철수";
let age = 25;
let isStudent = true;
let height = 175.5;

console.log("이름:", name);
console.log("나이:", age);
console.log("학생 여부:", isStudent);
console.log("키:", height);

브라우저 콘솔에서 위 코드를 실행해보세요!

2

값 변경해보기

// 값 변경
age = 26;
isStudent = false;

console.log("변경된 나이:", age);
console.log("변경된 학생 여부:", isStudent);

결과

콘솔에 개인 정보가 출력되고, 값이 변경되는 것을 확인할 수 있습니다!

실습 2: 계산기 만들기

1

계산 변수 만들기

// 숫자 변수
let num1 = 10;
let num2 = 5;

// 계산 결과 저장
let addition = num1 + num2;
let subtraction = num1 - num2;
let multiplication = num1 * num2;
let division = num1 / num2;

console.log(num1 + " + " + num2 + " = " + addition);
console.log(num1 + " - " + num2 + " = " + subtraction);
console.log(num1 + " × " + num2 + " = " + multiplication);
console.log(num1 + " ÷ " + num2 + " = " + division);

결과

계산 결과가 콘솔에 출력됩니다:

  • 10 + 5 = 15
  • 10 - 5 = 5
  • 10 × 5 = 50
  • 10 ÷ 5 = 2

실습 3: 템플릿 리터럴 사용하기

1

템플릿 리터럴로 문자열 만들기

// 변수 준비
let userName = "홍길동";
let userAge = 30;
let userCity = "서울";

// 기존 방식 (+ 연산자)
let oldWay = "안녕하세요! " + userName + "님, 나이는 " + userAge + "세이고 " + userCity + "에 살고 계시는군요!";

// 템플릿 리터럴 방식 (백틱 사용)
let newWay = `안녕하세요! ${userName}님, 나이는 ${userAge}세이고 ${userCity}에 살고 계시는군요!`;

console.log("기존 방식:", oldWay);
console.log("새로운 방식:", newWay);

// 계산도 가능
let message = `${userName}님은 ${2024 - userAge}년에 태어나셨습니다.`;
console.log(message);

결과

템플릿 리터럴을 사용하면 문자열 안에 변수를 쉽게 삽입할 수 있습니다!

💡 ${변수명} 형태로 사용하면 됩니다!

자주하는 실수와 해결법

❌ 실수 1

변수 선언 없이 사용하기

❌ 잘못된 코드
name = "홍길동";  // let이나 const 없이 사용
console.log(name);
✅ 올바른 코드
let name = "홍길동";  // let으로 선언
console.log(name);

해결법: 변수를 사용하기 전에 반드시 let이나 const로 선언하세요.

❌ 실수 2

const 변수 값 변경하기

❌ 잘못된 코드
const PI = 3.14;
PI = 3.14159;  // 오류 발생!
✅ 올바른 코드
let pi = 3.14;      // 변경 가능한 변수
pi = 3.14159;       // 정상 작동

// 또는
const PI = 3.14159; // 처음부터 정확한 값

해결법: 값이 바뀔 수 있으면 let, 고정값이면 const를 사용하세요.

❌ 실수 3

문자열과 숫자 혼동하기

❌ 잘못된 코드
let num1 = "10";    // 문자열
let num2 = "5";     // 문자열
let result = num1 + num2;
console.log(result); // "105" (문자열 연결)
✅ 올바른 코드
let num1 = 10;      // 숫자
let num2 = 5;       // 숫자
let result = num1 + num2;
console.log(result); // 15 (숫자 계산)

해결법: 숫자는 따옴표 없이, 문자는 따옴표로 감싸세요.

❌ 실수 4

변수명 규칙 위반하기

❌ 잘못된 코드
let 2name = "홍길동";     // 숫자로 시작
let user-name = "김철수";  // 하이픈 사용
let let = "변수";         // 예약어 사용
✅ 올바른 코드
let name2 = "홍길동";     // 숫자는 뒤에
let userName = "김철수";   // 카멜케이스 사용
let userVariable = "변수"; // 의미있는 이름

해결법: 영문자나 _로 시작하고, 의미있는 이름을 사용하세요.
